## YAPL - Security Policy

At YAPL, we take the security of our users and developers seriously. This document outlines the security policy for the YAPL project.

### Reporting Security Vulnerabilities

If you discover a security vulnerability in YAPL, please email us immediately at [yapl@iamseraph.com](mailto:yapl@iamseraph.com).

If you prefer to create an issue on GitHub, please do so in a private repository and invite the project maintainers.

When reporting a security vulnerability, please include as much information as possible, including:

* The affected version(s) of YAPL
* A description of the vulnerability
* Steps to reproduce the vulnerability
* The impact of the vulnerability
* Any potential remediation or mitigation strategies

We will acknowledge your report within 24 hours, and will work to address the vulnerability as quickly as possible.

### Response Timeline

We will acknowledge all security reports within 24 hours.

We will provide regular updates on our progress towards addressing the vulnerability, and will notify you when the vulnerability has been fixed.

### Disclosure Policy

We will work with you to coordinate the disclosure of the vulnerability, and will not disclose the vulnerability until it has been addressed.

We will credit you for reporting the vulnerability, unless you request to remain anonymous.

Thank you for helping to keep YAPL secure!
